# Transcript Skill 执行层面改进

## 背景

2026-05-15 处理《买方知识圈·非线性》(37,763 字) 时暴露四个执行层面问题：

1. 扫描 Agent prompt 漏替换文件路径，导致读了错误的文件
2. 扫描结果未校验，话题列表与文件内容不匹配未发现
3. 整理 Agent 输出格式不一致（纯 JSON / markdown 包裹 / 纯文本混用）
4. 终稿无质量抽查，不知道整理效果好坏

整体 pipeline 架构（扫描切分→并行整理→拼接）保留不变，仅修执行问题。

## 改进一：变量替换标准化

### 问题

当前 SKILL.md 中占位符混用三种写法：自然语言（"只用 Read 读一遍文件"）、花括号（`{path}`, `{title}`）、硬编码示例（切分命令）。容易漏替换。

### 方案

SKILL.md 顶部新增「变量替换清单」，全部标记统一为 `{VAR_NAME}` 大写花括号格式：

```
## 变量替换清单（执行前逐项检查）

| 标记 | 含义 | 出现步骤 |
|------|------|---------|
| `{INPUT_FILE}` | 规范化后的文件路径 | 扫描 Agent, 校验 Agent |
| `{NORMALIZED_FILE}` | 规范化脚本输入路径 | 第一步 |
| `{CHUNKS_DIR}` | 片段输出目录 | 第二步 |
| `{OUTPUT_DIR}` | 整理产出根目录 | 第二步/第四步 |
| `{SEGMENT_FILE}` | 单个片段文件路径 | 第三步 |
| `{SEGMENT_NUM}` | 片段编号(01-99) | 第三步 |
| `{TITLE}` | 话题标题 | 第三步 |
| `{EXPECTED_COUNT}` | 期望段数 | 第三步 |
| `{TOPIC_LIST}` | 扫描结果完整输出 | 第二步 |
```

各步骤正文中的占位符全部改为对应标记。

## 改进二：扫描结果 AI 校验

### 问题

扫描 Agent 可能读错文件或输出不相关话题，当前无任何校验。

### 方案

扫描 Agent 返回后、切分前，启动 Haiku 校验 Agent：

```
Read {INPUT_FILE} 的 1-10行、中间10行、末尾10行，了解内容主题。
已知扫描 Agent 对全文的话题切分结果为：
{TOPIC_LIST}

判断：话题标题能否合理对应这个文件的内容主题？
- 能 → 输出 "PASS"
- 明显跑偏（标题与内容无关）→ 输出 "FAIL|理由"

只输出一行。不要解释 PASS。
```

FAIL 时销毁结果、重扫一次。两次都 FAIL 则停下来让用户介入。

### 耗时

校验 Agent 只读 ~30 行 + 话题列表，Haiku 处理量远小于全文扫描，正常 ~15-20 秒。

## 改进三：整理 Agent 输出格式加固

### 问题

Prompt 写「回复只返回 JSON」，但 Agent 返回格式不一致：有的纯 JSON、有的 markdown 包裹、有的带解释文字。虽然都能手工处理，但不可靠。

### 方案

整理 Agent prompt 末尾的格式约束改为：

```
【输出格式】
仅输出一行合法 JSON 对象。不要 markdown 包裹、不要解释文字、不要问候语。

正确：
{"file":"{OUTPUT_DIR}/chunks/clean_{SEGMENT_NUM}.md"}

错误示例（以下全部禁止）：
  - ```json\n{"file":"..."}\n```
  - 已整理完成，文件保存在...
  - ## 整理结果\n...
  - 好的，我来处理这个片段。\n\n```json...
```

同时在 Step 3 完成后加脚本校验：`wc -c < clean_*.md > 100`，空文件或过短标记为 FAIL，只重跑缺失/异常片段。

## 改进四：终稿质量抽查

### 问题

拼接后的整理稿没有任何质量检查，不知道去噪效果、术语一致性等情况。

### 方案

Step 4 拼接完成后启动 Haiku 质检 Agent：

```
读 {整理稿全文}。

仅检查以下 4 项，不修改原文：
1. 跨段重复：相邻段落是否有内容重复（合并遗漏）
2. ASR 错字残留：是否有语义不通的字符
3. 术语一致性：同一概念是否有不同写法
4. 无关内容：是否有明显不属于访谈的文本混入

按项输出 PASS 或 FAIL|位置+示例。不修复。
```

发现问题不阻塞流程，把问题清单附在最终报告里供用户判断。

## 不改的部分

- 扫描+切分+全量并行 的 pipeline 架构
- 短文本快速通道逻辑（NEEDS_SPLIT=false）
- 速览稿生成逻辑
- scripts/split_by_lines.sh 和 text_stats.py 功能
